### Added

- `apply-review-disposition.md § Mandatory adherence` — explicit clause stating Steps 1–5 of the conservative author-side disposition are ALWAYS executed for every reviewer comment (inline or body); no ad-hoc path is permitted.
- Inline-thread reply instruction in `apply-review-disposition.md` — after producing the per-comment output (Step 5), the disposition now instructs replying to each review thread with its Nature / Severity / Decision + rationale.
- `apply-review-disposition.md § Step 6 — Reply and resolve on the thread` — Decision→thread-action mapping table: APPLIED → reply + `resolveReviewThread`; PARTIAL → reply + conditional resolve; DEFERRED/REJECTED/NEEDS-CLARIFICATION → reply, leave open. Pins "never mass-resolve", "never resolve unfinished work", and the resolve ≠ dismiss invariant (resolving a thread does NOT dismiss a `CHANGES_REQUESTED` review; re-review remains the reviewer's action).
- `DEFERRED` decision state added to the Decision enum — defined as agreement-plus-postponement (valid finding intentionally postponed to a tracked follow-up); distinguished from `REJECTED` (disagreement). A `DEFERRED` without a tracked follow-up reference (issue `#NNN`, backlog item, or explicit note) is not valid. Step-5 output template gains `Follow-up (if DEFERRED):` and `Thread action:` lines.
- Step-4 heading renamed to `## Step 4 — Resolve the concern, don't obey the instruction` — disambiguates concern-resolution (Step 4, about code) from GitHub thread-resolution (Step 6, about GitHub state).
- `gh-fallback.md § Tier B — list review threads (map comment → thread id)` — GraphQL `reviewThreads(first: 100)` listing query; documents that this is the only way to obtain thread IDs and `isResolved` state (REST API has no thread endpoint).
- `gh-fallback.md § Tier B — reply to a review thread` — primary path: GraphQL `addPullRequestReviewThreadReply`; curl fallback: REST `POST .../comments/{root_id}/replies`; operator-paste tier.
- `gh-fallback.md § Tier B — resolve a review thread` — GraphQL `resolveReviewThread` mutation; explicitly states NO REST equivalent exists; documents attempt-and-degrade-on-403 with `X-Accepted-GitHub-Permissions` diagnosis; repeats resolve ≠ dismiss invariant; rate-limit and idempotency notes.
