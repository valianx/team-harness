### Changed

- **Batch consolidation named as the explicit default** (issue #354): a same-repo task batch now ships as ONE consolidated PR by default — all task branches merge into `batch/<name>-verify`, the version bumps once, the changelog is one entry, and exactly one PR covers all batch work. This makes the existing Step 5 behavior an explicit named contract rather than a silent implementation detail. (`agents/orchestrator.md § Multi-Task Orchestration — Consolidation default`)
- **Operator opt-out for separate PRs**: the operator may request separate PRs by saying "keep them as separate PRs" / "separate PRs". The orchestrator never chooses separate PRs on its own authority. Only genuine blockers bypass this default without opt-out: (a) an unresolvable Step-5a merge conflict; (b) a reason from the plan-reviewer's closed list (`coexistence window`, `production signal`, `cross-repo deploy gate`).
- **Reconciled milestone wording**: the "one task = one PR" milestone rule is clarified to mean "a single task is never split across PRs" — the anti-split invariant — not "force each batched task into its own PR." The two rules (anti-split + consolidation default) are the same constraint read in opposite directions. (`CLAUDE.md §5`, `agents/ref-special-flows.md § Batch consolidation vs the anti-split invariant`)
- **Same-delivery-flow alignment** (AC-6): the consolidated batch ships via the same `delivery` agent (Step 5d) and the same review → merge → worktree-teardown lifecycle as a single task. No separate batch-delivery path. (`agents/orchestrator.md § Multi-Task Orchestration — Consolidation default`)
- **Plan-reviewer Rule 1 complementary clause**: Rule 1 ("one PR per service") now explicitly states it is the split-direction rule and is complementary to batch consolidation — a batch of independent tasks consolidating into one PR is not a Rule 1 split. (`agents/plan-reviewer.md § Rule 1`)
- **Suite 114** (`batch-consolidation-default`): structural assertions for the named consolidation default, the opt-out phrases, the genuine-blocker closed list, and the reconciled wording at all four surfaces. (`tests/test_agent_structure.py`)
