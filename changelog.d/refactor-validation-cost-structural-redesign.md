### Added

- `docs/verification-packet.md`: canonical schema for the shared Stage-2 verification packet (`00-verify-packet.md`) — header (tree anchor, base ref, packet version), scope flags, changed-files table, implementation summary, test artifact, full-document pointers, ≤120-line cap, staleness/rebuild triggers, an authority-scoping rule (the packet is a non-authoritative navigation digest — no verdict-bearing fact may rest solely on it; acceptance criteria are never copied into it and are always live-read from `01-plan.md § Task List`), preserved-reads floor per verifier, and a per-run, operator-evaluated parity line reported in every full-pipeline summary — dispatch denominator grounded in the workspace verdict docs (`00-subagent-trace.jsonl` breadcrumbs demoted to upward-only enrichment that can only add, never shrink, the count), a single dispatch-floor derivation from `00-state.md` scope flags (never from `§ Agent Results`), a three-bucket telemetry classification (missing telemetry always counts as fallback-signal, never acceptance), and `UNMEASURABLE` rendering below the floor. The line is a reporting aid the operator evaluates whenever desired; rollback is a one-line contract flip (the packet-first ladder default in the five verifier Session Context Protocols) shipped as a normal PR — no automatic trigger, run counter, or window exists.
- `agents/ref-intake-flows.md`: new on-demand reference file (mirrors `ref-special-flows.md`) holding the four conditional Phase 0a intake sub-flows — milestone continuity, initiative create-or-join, initiative detection + confirm, and language/english-learning-set intent handling — plus the ClickUp conversational-intents flow, each triggered from a short pointer left in `agents/orchestrator.md`.

### Changed

- **Shared verification packet (A3).** The orchestrator writes one `00-verify-packet.md` at Phase 2.7 close; the five Stage-2 verifiers (`qa`, `security`, `adversary`, `tester` run-only, `ux-reviewer` validate) switch their Session Context Protocol from a full workspace-doc read to a mandatory live read of the acceptance criteria (Step 0) plus a packet-first, depth-on-demand ladder for the narrative documents, with a mandatory integrity spot-check (tree anchor, changed-file existence) and a fail-open fallback to the full manifest read when the packet is absent or stale. Each verifier's preserved-reads floor (source-code/sketch/security-report reads) is written explicitly and stays untouched; source-scanning agents resolve their authoritative scan-target list from `git diff --name-only` against the packet's recorded base ref, not from the packet's changed-files table alone. New `packet_used`/`packet_escapes`/`packet_integrity` status-block fields propagate into the `phase.end.tools` telemetry.
- **Acceptance-checker narrowed to drift-only, scheduled concurrently with Build Verification (A4).** `agents/acceptance-checker.md`'s mandate shrinks to 5 retained catches nobody else owns (AC drift, per-delta spot-check, original-description-vs-delivered grounding against the §-scoped implementation record, surviving `[CONSTRAINT-DISCOVERED]` tags, implicit-NFR scan), with default reads capped at ≤3 files; per-AC satisfaction re-validation and the sketch surface-diff are dropped, each with a named owner (qa's verdict; qa's mandatory sketch cross-check). Phase 3.6 now dispatches in the SAME message as Phase 3.75's build/lint calls instead of running serially after them, with a conditional re-run rule and a Stage-3 toast that waits for both to complete.
- **Orchestrator boot reduction (A7).** The dedicated boot dispatch-probe subagent is removed — Task availability is verified by the run's first genuine dispatch instead, with identical Dispatch-blocked-exit semantics. The Phase 0a CLAUDE.md re-read is now conditional on the absence of the top-level session's `claudeMd` injection marker (mandatory only on subagent/takeover/compaction paths). Four conditional Phase 0a intake sub-flows move verbatim to the new `agents/ref-intake-flows.md`, replaced in the spine by 1-2 line triggers; tier tables, the security-sensitive path list, and the Discover/checkpoint (B1/B2/B3) machinery are unchanged.
- **Observability compliance and calibration (A8).** Marking a Phase Checklist item `[x]` and appending its `phase.end` event are now one inseparable step. A stage-gate reconciliation backstop (STAGE-GATE-1/2/3) backfills missing `phase.end` events from `00-subagent-trace.jsonl` breadcrumb pairs, tagged `tokens_estimated: true` and `backfilled: true`, never overwriting a measured event. `## Cost` emission in `00-pipeline-summary.md` drops from an unenforced "every transition" rewrite to 4 mandatory checkpoints (STAGE-GATE-1, Stage-2 close, every `iteration.start`, `pipeline.complete`) with a fail-closed sanity-check assert. Six stale cost estimates (e.g. ratify-plan "~3-5K tokens") are replaced with measured June 2026 figures or explicit `(estimate)` tags, and every stated cost figure now carries a source tag going forward.
