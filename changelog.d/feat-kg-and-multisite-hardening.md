### Added

- **A1 — Multi-site invariant enumeration (architect + delivery):** `agents/architect.md` now requires a `### Multi-site invariants` table in `01-plan.md § Review Summary` whenever an invariant lives in more than one file, enumerating every site and fencing sites that must not change. A new `agents/delivery.md` Step 9.4a (pre-STAGE-GATE-3) reads that table and verifies all declared sites match each other, returning `status: failed` with a "partial sync" report on any divergence. The existing Step 9.0 enumerated version-site table and its FENCED-OFF block are preserved unchanged as the worked example.
- **A2 — Anchor-scoped assertion canonical principle + meta-test (Suite 126):** `docs/testing.md` gains a new `### (iii) Structural assertions are anchor-scoped, never whole-file first-occurrence` principle that canonicalizes `_slice_section` as the required assertion shape and bans six fragile patterns (fixed-width `find()+N` content slices, whole-file substring checks for structural assertions, exact-version string pins, `changelog.d/` file-existence assertions). Suite 126 (`anchor-scoped-assertion-meta-test`) enforces this principle: it hard-fails on any matching fragile assertion in `tests/*.py`, passes on the refactored tree with no grandfather allowlist, and includes a synthetic-positive check to prove RED/GREEN behaviour. All pre-existing fragile `find()+N` content-slice assertions in `tests/test_agent_structure.py` have been refactored to `_slice_section` (zero of the banned shape remain). Suite 126 is registered in `docs/testing.md § Named suites` and carries the self-referential guard.
- **B1 — Defect-aware second KG query + `kg_hit_used` recall field (closes #403):** `agents/orchestrator.md` defines a best-effort, non-blocking second `mcp__memory__search_nodes` query at Phase 1 end / Phase 1.5 entry, seeded from the located files and failure mode, appended as a new block to `00-knowledge-context.md`; on MCP error it logs `KG: unavailable` and continues. Every leaf-agent Return Protocol status block (`architect`, `implementer`, `tester`, `qa`, `security`, `delivery`) now declares a `kg_hit_used: [node-name, ...]` field, and `agents/orchestrator.md` documents aggregating it into the `phase.end` tools object to make KG recall measurable in the pipeline trace.
