### Changed
- **BREAKING:** Claude Code hook gate logic is cut over from Bash to compiled TypeScript (#446). `.claude-plugin/hooks.json` now wires every gate through `hooks/run-ts-hook.sh`, a fail-closed launcher with no gate logic of its own that execs `node` against the same compiled `hooks/ts/dist/*.cjs` artifacts opencode already consumed — one source of gate logic for both runtimes. The 11 Bash hook bodies wired in CC (`policy-block.sh`, `dev-guard.sh`, `gcp-guard.sh`, `worktree-guard.sh`, `prepublish-guard.sh`, `checkpoint-guard.sh`, `session-start.sh`, `language-user-prompt.sh`, `subagent-trace.sh`, `precompact-snapshot.sh`, `notify-stage.sh`) plus `_json-extract.sh`, `_hook-profile.sh`, and `hooks/config.json` are retired; only `hooks/sketch-guard.sh` (not an event hook) remains Bash. Native per-OS desktop notification is ported into `notify-stage.cc.ts` — no functional loss. Measured latency improved 635ms → 287ms aggregate across the 5 Bash-matcher gates (no combined dispatcher needed).
- **BREAKING:** the Go installer's Claude Code install path is retired (#446, operator directive). The binary is opencode-only; a bare CC invocation now prints a redirect notice to `/plugin marketplace add valianx/team-harness`. The marketplace plugin is the only Claude Code install/update channel going forward.
- Functional test suites for the 6 security floors and session-start/language-user-prompt are re-pointed from the Bash|TS dual-target to the TS artifact as the sole target (`tests/run-all.sh`); no suite lost coverage — a previously-orphaned `test_prepublish_guard.sh` was retargeted and rewired, restoring its unique coverage.

### Security
- Fail-closed launcher (`hooks/run-ts-hook.sh`) hardened to deny when a deny-floor `.cjs` artifact is corrupt, empty, or crashes without emitting a decision — not only when the artifact or `node` runtime is absent (SEC-PR2-001, Critical, fixed and re-verified; new regression suite, 34/34).
- `gcp-guard`'s catastrophic-verb raw-payload fail-safe made reachable again on malformed/command-absent payloads, closing a coverage gap left by the TS port (SEC-PR2-002, Medium, fixed and re-verified).
- Post-flip hardening folded into the same change: `policy-block` curl secret-scan predicate broadened (`-d`, `-H 'Authorization: Bearer'`, `--json`, `-F`) with the empty-stdin vs. non-empty-unparseable fail-open/fail-closed split restored; `prepublish-guard` now evaluates rename source paths (not just destination) against the shipped-asset floor, and the opencode entry gains the `#411` cwd-scoping fix; `worktree-guard`'s raw-payload scan is gated to the fallback path only, eliminating a spurious advisory trigger; `subagent-trace` breadcrumbs restore the `agent_id` correlation key (SEC-DR-007) dropped during the initial TS port.
