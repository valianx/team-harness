### Added

- Opt-in english-learning correction mode: new `english_learning` boolean key in `~/.claude/.team-harness.json`; when `true`, the session-start hook injects a behavioral directive that gives the operator a brief, low-key English correction signal at the start of each reply — a plain-ASCII `:)` for already-correct messages, a compact minimal-edit correction block for correctable errors. Mode is off by default (absent key = off). Configurable via `/th:setup` Step 3.6 (`[y/N]` prompt) or via a chat toggle with a persistence marker routed through the orchestrator's Y/n confirmation gate; a toggle without a persistence marker applies as a session-only override. New `load_english_learning` function added as Load 3 in `hooks/session-start.sh` (after `load_language`, before `load_workspace_mode`); orchestrator Step 6 intent rows `(b′)` and `(c′)` handle persistent-set and session-toggle respectively; the `orchestrator-dispatch-rule.md` managed block (synced into `~/.claude/CLAUDE.md` by `/th:setup` and `/th:update`) documents the propagation contract.
