### Added
- opt-in, additive per-provider cost tiering for opencode agents (issue #424): a curated `providerTierFamily`/`providerTierConcrete` map (Anthropic-only at launch, provider-generic architecture) lets the operator bake a concrete `model:` id per agent — derived from each agent's CC source tier (opus/sonnet/haiku) — for one selected provider, via the new `--opencode-tier <provider>` installer flag or the persisted `opencode.cost_tier_provider` config key. Absent selection leaves the existing model-less baseline unchanged. The curated map is locked byte-identical across `cmd/install/transform.go`, `tools/harness-migrate/migrate.mjs`, and `skills/update-models/SKILL.md` by a new structural parity test (`cmd/install/tier_test.go`).

### Fixed
- `skills/update-models/SKILL.md`: the resolver was reading the wrong models.dev API shape — it iterated the top level as if keyed by model id, but the live API is keyed by provider with a nested `models` object grouped by the model's `family` field. This made the resolver a silent no-op against the live API. Corrected to the real nested shape, with a regression case proving the old flattened shape no longer resolves anything, and added the never-mix-providers regeneration rule (a provider switch regenerates the whole installed set).
