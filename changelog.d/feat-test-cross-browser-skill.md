### Added

- `/th:test-cross-browser`: new on-demand skill that routes to the orchestrator with `Mode: test` + `cross_browser: true`, triggering the tester's cross-engine matrix (Chromium / Firefox / WebKit + branded channels + optional cloud grid). The skill reuses `/th:test`'s frontend-marker detection and adds `frontend_scope: true` when markers match. It is the **only** producer of `cross_browser: true`; the normal `/th:test` flow and all default pipeline paths are byte-for-byte unchanged.
- `agents/testing-refs/cross-browser.md`: new axis reference loaded alongside the warranted `e2e`/`browser-mode` reference only when `cross_browser: true` is present. Documents the honest browser model (3 native engines, verified branded Chromium channels, popular-browser → engine → how-to-run table with Brave/Opera/Vivaldi caveats, cloud device grid via `connectOptions.wsEndpoint`, device emulation), per-engine skip annotations (`test.skip`/`test.fixme`), the browser × resolution matrix strategy, the third-party UI library review step, the 13-family known-failure-modes catalog (form controls, floating UI/scrolling, layout/stacking, typography, CSS feature-support gaps, JS APIs, media/animation/input, forced colors, dark mode, RTL/bidi, print/paged media, Shadow DOM/Web Components, canvas/WebGL), and CI guidance defaulting to chromium-only on PR with the full matrix gated behind a `cross-browser` label or schedule.
- `agents/testing-refs/_index.md`: added `cross-browser` router row (stacks covered: react-nextjs; nestjs/go/python: n/a) with a note that the file loads alongside a warranted `e2e`/`browser-mode` reference and only when `cross_browser: true` is set.
- `agents/tester.md`: added a flag-gated Phase-0 clause that loads `agents/testing-refs/cross-browser.md` in addition to the warranted e2e/browser-mode reference when `cross_browser: true` is present in the payload; absent the flag, tester behavior is identical to today.
