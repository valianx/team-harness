### Fixed
- Delivery's Step 11.4 no longer asserts universal CodeRabbit presence — a three-signal detection rule (`coderabbit_configured` boot-time hint, repo-root `.coderabbit.yaml`/`.yml`, or a positive `CodeRabbit` entry in the already-fetched `statusCheckRollup`) gates the review-surface framing, so consumer repos without CodeRabbit are no longer told to wait for a check that never appears. The mechanical `ci_state` rules and `coderabbit_configured` boot-time recording (`agents/orchestrator.md` Phase 0a Step 7) are unchanged/added respectively; zero new GitHub API calls.
- The tester's `module-test` embedded security mini-scan is documented as the Test Pipeline Flow's **sole** security layer — that flow dispatches no dedicated security agent, so the mini-scan runs unconditionally unless the operator passes `--skip-security`. The operator opt-out is hardened to exact-literal `true` parsing, and a skip is recorded in `03-testing.md § Security Findings` stating the module ran with no security layer. (A prior iteration of this fix introduced a `security_agent_will_run` conditional skip and a compensating Phase 3.5 cross-check premised on a same-run security dispatch coexisting with `module-test`; that premise does not hold anywhere in the plugin today, so the flag, its threading, its status-block field, and the cross-check were removed rather than shipped as dead wiring. A structural test guards their absence.)
