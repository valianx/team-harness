- [pattern] Self-describing task list: `02-task-list.md` carries a `Status:` field per PR (`pending | in-progress | verified | merged | blocked`) and AC checkboxes flipped to `- [x]` on PASS — readers don't have to cross-reference `00-state.md` or `04-validation.md` to know progress.
- [pattern] Bidirectional cross-references for split PRs: base PR carries `Cleanup PR:` with rationale (what stays alive, condition for cleanup merge, risk if cleaned together); secondary PR carries `Base PR:` back-reference. The closed-list label `Split reason: coexistence window` alone is not enough.
- [pattern] Phase 1.6 plan-review is inviolable: `01-plan-review.md` must exist with a `## Verdict` line before STAGE-GATE-1 is emitted to the user. When the harness refuses a nested `Task(subagent_type='plan-reviewer')`, the orchestrator runs the 5-rule audit inline using `agents/plan-reviewer.md` as the procedure spec — never escalates to the user without the audit. Subagent and inline executions share the same max-3 iteration budget.
- [constraint] Post-STAGE-GATE-1 writes to `02-task-list.md` are limited to: the `Status:` field on a PR header (orchestrator transitions to in-progress/verified/blocked; delivery transitions to merged) and AC checkbox flips `- [ ]` → `- [x]` by qa on PASS. Files, AC text, dependencies, Cleanup PR / Base PR, Title, Branch, Notes are frozen.
- [constraint] Agent frontmatter (`agents/*.md`) MUST be valid YAML. An unquoted `": "` (colon-space) inside a `description:` value silently breaks parsing and Claude Code drops the agent from the registered `subagent_type` list with no error surfaced. `tests/test_agent_frontmatter.py` enforces this — runs via `uv run --with PyYAML python` and parses every agent file.
- [stack] As of feat/go-installer, the installer binary is Go (stdlib-only, CGO_ENABLED=0, cross-compiled). `bin/install.py` is deprecated (one-release fallback). Bootstrap wrappers (`bin/install.sh`, `bin/install.ps1`) download the correct binary from the latest GH Release — they no longer require `uv` or Python on the host.
- [pattern] KG passive capture on delivery: the `delivery` agent (Step 11.5 of `agents/delivery.md`) calls Memory MCP `create_nodes` once per completed task with `nodeType: "process-insight"`, synthesising one node from the workspaces + CHANGELOG entry. The call is best-effort — if the Memory MCP server is unreachable or no reusable learning surfaced, the step logs the skip and continues. This builds the team's institutional knowledge automatically without operator curation. v0.5.0+.
- [decision] Hook notifications fire **only when the user must take an action** (default preset `ultra-quiet` in `hooks/config.json`): `Notification` matcher `idle_prompt` + `PreToolUse` policy gate, nothing else. The previous `permission_prompt` matcher and blanket `PostToolUseFailure *` were removed because they fire while work continues (especially under `skipDangerousModePermissionPrompt: true` and during MCP flutter) and accumulate as look-alike toasts in the Windows Action Center / system tray without action required. `hooks/README.md` exposes louder presets (`default-quiet`, `noisy`) for users who want them — but the default is silence-by-default. Drives the "notifications must be actionable" contract for the whole `hooks/` distribution.
- [decision] Curl one-liner install via `go:embed` + GitHub Pages: installer binary is self-contained (`//go:embed agents skills hooks` at repo root level — embed paths must be in/below the source file's directory); bootstrap scripts (`bin/install.{sh,ps1,cmd}`) use the deterministic `releases/latest/download/` URL without calling the GitHub API (removes rate-limit exposure + brittle JSON parse); scripts are published to `https://valianx.github.io/team-harness/install.{sh,ps1,cmd}` via `.github/workflows/pages.yml` triggered on `release: published` only. Contributors testing local edits use `go run ./cmd/install` — `//go:embed` snapshots working-tree bytes at compile time.
