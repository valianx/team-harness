# Project Structure Convention

This document defines the file organization for generated interactive presentations.

---

## Directory Layout

```
{project-name}/
├── package.json                 ← Dependencies and scripts
├── vite.config.ts               ← Vite build config
├── tailwind.config.ts           ← Tailwind theme (scene colors, fonts)
├── postcss.config.js            ← PostCSS with Tailwind + autoprefixer
├── tsconfig.json                ← TypeScript strict config
├── index.html                   ← Entry point with font links
├── dist/                        ← Build output (gitignored, generated by `npm run build`)
└── src/
    ├── main.tsx                 ← React root mount
    ├── App.tsx                  ← Presentation shell (navigation, keyboard, scene routing)
    ├── types.ts                 ← Scene type definitions (FlowSceneData, AnimatedSceneData, SlideSceneData)
    ├── assets/
    │   └── slides/              ← Imported slide images (from-ppt mode)
    │       ├── slide-1.png
    │       ├── slide-2.png
    │       └── ...
    ├── styles/
    │   └── globals.css          ← Tailwind directives + React Flow overrides + nav styles
    ├── hooks/
    │   └── useScene.ts          ← Scene navigation state (next/prev/goTo)
    ├── components/
    │   ├── Scene.tsx            ← Scene type router (switch on scene.type)
    │   ├── FlowScene.tsx        ← React Flow wrapper (highlight steps, tooltips)
    │   ├── AnimatedScene.tsx    ← GSAP timeline wrapper (play/pause/restart controls)
    │   ├── SlideScene.tsx       ← Static HTML slide (dangerouslySetInnerHTML)
    │   └── scenes/              ← Generated scene-specific components
    │       ├── DataFlowDiagram.tsx     ← Example: custom React Flow graph
    │       ├── DeployAnimation.tsx     ← Example: GSAP-animated deployment flow
    │       └── ArchitectureSVG.tsx     ← Example: SVG composition
    └── data/
        └── presentation.ts      ← Scene configuration (PresentationConfig object)
```

---

## File Responsibilities

### Shell Files (Template — Rarely Modified)

| File | Responsibility | When to modify |
|------|---------------|----------------|
| `main.tsx` | React root mount | Never |
| `App.tsx` | Navigation shell, keyboard handling, scene rendering | Add new navigation features (e.g., fullscreen toggle) |
| `types.ts` | Scene type definitions | Add new scene types |
| `hooks/useScene.ts` | Navigation state | Add scene-level features (e.g., sub-steps within a scene) |
| `components/Scene.tsx` | Type router | Add case for new scene types |
| `components/FlowScene.tsx` | React Flow container | Customize flow behavior globally |
| `components/AnimatedScene.tsx` | GSAP container + controls | Customize animation transport globally |
| `components/SlideScene.tsx` | Static HTML container | Customize slide styling globally |

### Generated Files (Per Scene — Agent Creates These)

| File | Responsibility | When created |
|------|---------------|--------------|
| `components/scenes/{SceneName}.tsx` | Scene-specific React component | When a scene needs custom rendering beyond the type defaults |
| `data/presentation.ts` | Full scene configuration | On `new` mode; extended on `add-scene` and `from-ppt` modes |
| `assets/slides/slide-{N}.{ext}` | Static slide images imported from an existing deck | On `from-ppt` mode, for slides rendered as embedded images |

### Slide Assets Convention (`from-ppt` mode)

Static slide images are stored in `src/assets/slides/` with the naming convention `slide-{N}.{ext}`:

- **`{N}`** — 1-indexed integer matching the original deck order (e.g., `slide-1.png`, `slide-4.jpg`)
- **`{ext}`** — original file extension preserved (`png` or `jpg`)
- Images are imported via standard Vite asset handling (`/src/assets/slides/slide-1.png`) — Vite processes them through the asset pipeline (hashing, optimization) at build time
- Only slides that use the "embed as background image" strategy get saved here — slides recreated in HTML/CSS/SVG have no corresponding asset file
- The `assets/slides/` directory is created only when at least one slide uses the embed strategy

## Scene Configuration Pattern

The presentation is configured in `src/data/presentation.ts` (or directly in `App.tsx` for simple presentations):

```tsx
import type { PresentationConfig } from "../types";

export const presentation: PresentationConfig = {
  title: "System Architecture Deep Dive",
  description: "Interactive walkthrough of the payment service",
  scenes: [
    {
      id: "intro",
      type: "slide",
      title: "Payment Service Architecture",
      subtitle: "How money flows through our system",
      content: `<div class="...">...</div>`,
    },
    {
      id: "overview-flow",
      type: "flow",
      title: "High-Level Flow",
      subtitle: "Click nodes to step through the process",
      nodes: [...],
      edges: [...],
      highlightSteps: [["client"], ["gateway", "client"], ["gateway", "payment-api"]],
    },
    {
      id: "processing-animation",
      type: "animated",
      title: "Transaction Processing",
      subtitle: "Watch how a payment is processed in real-time",
      component: "TransactionAnimation",
    },
  ],
};
```

## Naming Conventions

| Item | Convention | Example |
|------|-----------|---------|
| Scene IDs | `kebab-case` | `payment-flow`, `auth-sequence` |
| Component files | `PascalCase.tsx` | `PaymentFlow.tsx`, `AuthSequence.tsx` |
| CSS classes | Tailwind utilities (no custom CSS files per scene) | `className="text-scene-accent"` |
| Node IDs in React Flow | `kebab-case` | `api-gateway`, `auth-service` |
| GSAP target selectors | `.kebab-class` or `#unique-id` | `.flow-arrow`, `#step-3-label` |

## Adding a Scene (add-scene mode)

When the agent adds a scene to an existing project:

1. Create `src/components/scenes/{SceneName}.tsx` if the scene needs custom rendering.
2. Update `src/data/presentation.ts` — add the new `SceneData` entry to the `scenes` array.
3. If a new custom node type is needed for React Flow, add it to `FlowScene.tsx` `nodeTypes`.
4. If a new React import is needed, ensure `package.json` has it (run `npm install`).
5. Validate: `npm run build` must succeed.

## Build and Preview

```bash
# Install dependencies
npm install

# Development server (hot reload)
npm run dev

# Production build (outputs to dist/)
npm run build

# Preview production build locally
npm run preview
```

The `dist/` folder contains a self-contained static site. It can be served from any HTTP server, deployed to Vercel/Netlify/GitHub Pages, or opened locally.
